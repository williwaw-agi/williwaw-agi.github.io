Williwaw*

Williwaw will you take you on a virtual adventure, on a sailing boat across the sea. Get on board and control the elements in order to sail where you have never sailed before. Blow into your phone to create the wind in the sails and tilt your phone to steer the ship's wheel. Williwaw allows a number of adventurers to sail the virtual sea simultaneously, using multiple mobile phone devices. Beware of the other adventurers on the seas! Canon balls will soon start flying.

The Making Of

Unity

The virtual game environment was created using Unity.

Android

The current application was designed to run on Android.

GitHub

Code hosting and version control was accomplished using GitHub.

Virtual Environment


The Packet Manager was used to import Terrain Tools. The Unity Asset Store was used to download and import ready-made Assets such as trees. A Terrain object was created using Terrain Toolbox. Then, Unity’s built-in Terrain Tools was used to shape the terrain using a variety of brushes to raise/lower terrain, smooth height, paint terrain, paint trees and paint details (such as grass and mushrooms). Assets were imported to create a custom skybox and to place clouds in the sky, and a custom script was then created to have the clouds move across the sky in-game. Lastly, an audio clip was imported and attached to the boat camera as an audio source, in order to simulate ambient sounds.

Interaction


There exist several forms of interaction in this application. Using the input of the phone’s accelerometer, the direction of the ship can be controlled. It allows players to tilt the phone in order to steer the boat left and right. The accelerometer has some smoothing applied to it, using Unity’s Lerp function, in order to reduce jitter. The microphone’s sound input is used to create wind in the game. A low pass filter is applied to reduce unwanted frequencies affecting the input. The phone’s light sensor is used for the spawning of clouds. The sensor measures the current lux of the ambient lighting in the room. By covering up the sensor, and thus lowering the light-level, the player is able to spawn clouds above their ship. The clouds follow the wind direction and can be used to hide the boat on the map. A wind effect was created using Unity’s particle system that responds to a wind zone.

Networking


Networking was vital to enable multi device support in the current game. One challenge was that the users were to each have an asymmetrical view of the world. The goal was to enable both a first-person camera view through each player’s client device and at the same time allow the server device to project an overview on a large screen. All camera views had to be in sync at all times. In order to accomplish this, Unet HLAPI was used to set-up a custom networking system. It allows for easy synchronisation of game object states between devices.
